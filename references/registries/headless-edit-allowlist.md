# Headless Edit Allowlist Registry

Manually maintained. Do NOT confuse with `permission-bypass-registry.md`,
which is auto-generated by `npm run audit:permissions` and tracks a
different concern (bypass-pattern occurrences in code).

This registry documents every path that `.claude/settings.json`
`permissions.allow` exposes to `Edit(...)` in headless
`claude -p --permission-mode acceptEdits` dispatches (Issue #127).

Every row must be justified by an audit against the canonical safety
rule from Issue #127: the target file must have **no runtime logic
that could be weaponized by a malicious edit** (no `exec`, no `spawn`,
no `fetch`, no `fs.write` to anywhere other than stdout). Pure
`process.stdout.write` reminder hooks qualify. Any file that writes
to the filesystem, spawns a subprocess, or performs I/O other than
stdout does NOT qualify, even if it is only a marker file.

Hooks that enforce invariants (pre-commit blocks, Stop blocks,
SessionStart blocks) MUST NOT be allowlisted regardless of their
runtime shape; human review at those chokepoints is load-bearing.

Drift between this registry and `.claude/settings.json` is enforced by
`web/test/headless-allowlist.test.ts`.
