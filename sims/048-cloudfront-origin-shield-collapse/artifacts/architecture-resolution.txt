  Failure chain:
    1. Lambda@Edge sets X-PrismStream-DeviceClass on viewer-request
    2. Cache policy includes that header in HeadersConfig (in the cache key)
    3. Cache key fragments into 7 slots per segment, hit rate collapses
    4. Origin shield cannot collapse requests with different keys
    5. S3 prefix request rate exceeds 5,500 GET/s, returns 503 SlowDown
    6. CloudFront returns 5xx to viewers

  Fix: remove X-PrismStream-DeviceClass from cache policy HeadersConfig.
  Move it to an origin request policy so it still reaches origin without
  fragmenting the cache key.
