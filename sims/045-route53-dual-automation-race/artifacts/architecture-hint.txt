  Two automations write to api.halcyonpay.com RRset:

  1. halcyon-health-check-lambda
     -> trigger: EventBridge schedule, every 30 seconds
     -> reads current RRset, queries NLB target health for each IP
     -> removes unhealthy IPs, adds newly-healthy IPs
     -> calls Route 53 ChangeResourceRecordSets (Upsert)
     -> NO SetIdentifier, NO version check

  2. halcyon-deploy-hook (CodeDeploy AfterAllowTraffic)
     -> trigger: every ECS deployment lifecycle event
     -> reads current RRset, removes old task IPs, adds new task IPs
     -> calls Route 53 ChangeResourceRecordSets (Upsert)
     -> NO SetIdentifier, NO version check
